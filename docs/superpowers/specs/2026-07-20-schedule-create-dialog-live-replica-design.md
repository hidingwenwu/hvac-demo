# 日程创建弹窗现网复刻设计

## 目标

将 `pages/ctrl-schedule.html` 的创建任务交互严格复刻为测试平台 `http://test-m.achelp.cn/task/index` 当前实现，并在现网表单校验通过后接入新的精确冲突提示。

## 现网采集基准

- 测试项目：产品部测试-按小时预付费。
- 采集日期：2026-07-20。
- 创建弹窗宽度约 `920px`，在桌面视口内居中，遮罩层整体允许纵向滚动。
- 弹窗采用暗色 Element UI 风格，标题为“创建任务”，右上角关闭，底部固定“取消 / 保存”。
- 页面左侧建筑树仅用于筛选当前范围内已有的日程任务，与创建弹窗完全解耦。

## 弹窗布局

弹窗主体为双栏布局：

- 左栏宽约 `222px`。
  - 选择群组：下拉输入，当前测试项目无群组时显示“无数据”。
  - 选择房间：楼栋、楼层、房间三级复选树。
- 右栏从弹窗左侧约 `272px` 开始，主要表单控件宽 `300px`。
  - 任务名称：必填，最大 30 字，实时显示 `0/30`。
  - 任务描述：可选，最大 100 字，实时显示 `0/100`。
  - 执行日期：必填日期范围。
  - 执行时间：小时与分钟两个独立下拉框；小时为 `00-23`，分钟为 `00-59`。
  - 指令动作：关机、关机并锁定、解锁开关、开机、解锁并开机、锁定设定。测试项目开启对应项目偏好后显示“锁定设定”。
  - 周运行时间：周一至周日复选框，以及“工作日 / 周末 / 取消”快捷操作。
  - 排除日期：日期选择、“排除”、“排除节假日”和排除日期列表。

## 房间范围

- 页面左侧未选择建筑范围时，展示全部已有日程任务；选择整栋、楼层或单个房间时，仅筛选命中该范围的已有任务。
- 无论页面左侧是否选择范围，都可以打开创建弹窗。
- 新建任务时，弹窗房间树不继承页面左侧选择，默认不勾选任何房间。
- 用户必须在创建弹窗内选择建筑物、楼层或房间；未选择房间时保存校验提示“请选择房间”。
- 保存、任务数据和冲突检测一律使用弹窗最终勾选房间。

## 指令动作联动

- 仅选择“开机”时显示空调设置区。
- 空调设置区包含：
  - 16-31°C 温度滑块、减号、加号和大号温度值。
  - 制冷、制热、送风、除湿四种模式卡片。
  - 高风、中风、低风三种风速卡片。
  - “取消选中”按钮，用于清除空调设置项。
- 关机、关机并锁定、解锁开关、解锁并开机不显示空调设置区。
- 选择“锁定设定”时显示独立锁定面板：
  - 是否开关锁定；勾选后必须选择“锁定开机”或“锁定关机”。
  - 是否模式锁定；勾选后必须至少选择制冷、制热、送风、除湿中的一种。
  - 是否温度锁定；勾选后显示 16-31°C 的温度上限和温度下限。
- “锁定设定”至少勾选一个锁定项；切换到其他动作再返回时，按现网行为清空锁定设置。

## 日期与星期联动

- 未选择完整执行日期范围时，周一至周日不可选。
- 选择日期范围后，只启用该范围内实际出现的星期。
- “工作日”选择当前已启用的周一至周五。
- “周末”选择当前已启用的周六、周日。
- 周运行时间区域内的“取消”清空星期选择。
- 周运行时间不是必填项；未选星期时仍可保存。

## 校验与保存流程

保存时按现网顺序校验：

1. 任务名称，提示“请填写任务名称”。
2. 执行日期，提示“请选择执行日期”。
3. 执行时间小时和分钟，分别提示缺失项。
4. 房间，提示“请选择房间”。
5. 选择开机时必须选择空调模式；风速可不选，未选择时保存为 `-`。
6. 选择锁定设定时，依次校验至少一个锁定项、开关锁定子项和模式锁定子项。

校验通过后再执行冲突检测。编辑任务继续沿用现有不检测规则。

## 精确冲突检测

- 只比较完全相同的执行时间点，弃用 30 分钟区间逻辑。
- 有效日期范围必须有交集。
- 交集内至少有一个自然日同时满足两个任务的周运行规则。
- 未选择周运行时间时，视为日期范围内每天执行。
- 弹窗最终房间与已有任务房间必须有交集。
- 任意任务、任意一天、任意房间命中即弹出一次提示，并汇总去重后的全部冲突房间。

提示文案：

`您当前设置的时间已存在其他日程任务，继续保存可能出现空调控制指令冲突，涉及房间[xxx.xxx.xxx...]。`

`请核对后确认：是否仍要继续保存当前日程任务？`

按钮：`取消 / 继续保存`

- 取消：只关闭提示，保留创建弹窗全部内容。
- 继续保存：只保存一次并关闭创建弹窗。

## 数据与代码边界

- `pages/ctrl-schedule.html`：页面结构、左侧已有任务筛选、真实弹窗布局、表单状态和保存流程。
- `pages/ctrl-schedule-form.js`：建筑范围解析、已有任务筛选、日期星期联动和表单校验纯函数。
- `pages/ctrl-schedule-conflict.js`：日期、星期、时间和房间冲突纯函数。
- `testcase/verify-schedule-conflict.cjs`：冲突规则、页面契约和关键交互状态测试。
- 保持 `_base.css` 与 `_ui.js` 的共享行为不变；本次所需复刻样式优先局部写入日程页面，避免影响其他原型。

## 验收标准

- 创建弹窗的尺寸、双栏结构、字段顺序、控件宽度、颜色、间距、按钮和滚动方式与测试平台一致。
- 页面左侧建筑树只筛选已有任务；创建弹窗默认无房间选中，弹窗最终勾选范围决定保存和冲突检测。
- 六种动作、“开机”条件区和“锁定设定”条件区行为与测试平台一致。
- 字数统计、日期星期联动、排除日期和校验提示可交互。
- 新冲突提示使用最新文案和按钮；取消保留数据，继续保存不重复新增。
- 自动测试通过，浏览器控制台无错误，并完成本地原型与测试平台的并排视觉核对。
